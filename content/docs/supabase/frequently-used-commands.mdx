## Installation

Install the Supabase CLI globally:

```bash
npm install -g supabase
# or
brew install supabase/tap/supabase
```

For more options, see the [official documentation](https://supabase.com/docs/guides/cli).

## Project Initialization

### Initialize a Supabase project

```bash
supabase init
```

This creates:

```
supabase/
  migrations/
  config.toml
```

### Link to a remote Supabase project

```bash
supabase link --project-ref <project-ref>
```

Find `project-ref` in: **Supabase Dashboard → Settings → General**

## Local Development

### Start local Supabase stack




## Database Commands

### Push migrations to remote

```bash
supabase db push
```

Applies all pending migrations to remote.

### Pull remote schema into migration

```bash
supabase db pull
```

Creates a new migration file from remote state.

### Diff local vs remote

```bash
supabase db diff -f update_supporters_table_with_is_monthly_and_metadata_entry
```

### Dump remote database

```bash
supabase db dump
```

### Execute SQL file

```bash
supabase db execute --file ./file.sql
```

## Migration Commands

### Create a new migration

```bash
supabase migration new <name>
```

Example:

```bash
supabase migration new create_users_table
```

### List all migrations

```bash
supabase migration list
```

Shows:
- Local migrations
- Remote applied migrations

### Repair migration status

Mark as applied:

```bash
supabase migration repair --local --status applied <timestamp>
```

Mark as reverted:

```bash
supabase migration repair --local --status reverted <timestamp>
```

> **Tip:** Always use `--local` flag when working with a local development environment to avoid applying changes to the remote database.
>
> **Warning:** Running without `--local` will apply changes to the remote Supabase project.

## Type Generation

Generate TypeScript types from your database schema.

### Local generation

Generate types from your local database schema:

```bash
supabase gen types typescript --local --schema public > ./src/supabase/types/supabase.ts
```

### Generate types for different projects

**Local schema:**

| Target | Command |
|--------|---------|

## Seed Commands

Database seeding using Snaplet.

### Initialize seed configuration

```bash
npx @snaplet/seed init
```

### Sync seed configuration

```bash
npx @snaplet/seed sync
```

### Prepare seed SQL

```bash
bun run seed.ts > supabase/seeds/seed.sql
```

### Reset and seed database

```bash
supabase db reset && bun run supabase/seeds/init.ts
```

## Typical Migration Workflow

```bash
# 1. Create a new migration
supabase migration new add_orders_table

# 2. Edit the SQL migration file

# 3. Push to remote
supabase db push
```

If remote schema was changed manually (via Dashboard):

```bash
supabase db pull
```

If migration history got cleared:

```bash
supabase migration repair --local --status applied <timestamp>
```

> **Warning:** Always use `--local` flag. Running without it will apply changes to the remote Supabase project.

## Important Notes

- `db push` applies ALL pending migrations in order.
- Supabase tracks migrations in:
  ```
  supabase_migrations.schema_migrations
  ```
- `migration repair` only changes tracking — it does not modify the actual schema.
